# 🐭 老鼠迷宮大冒險 — 技能說明文件

本文件說明此專案涵蓋的**學生學習技能**、**教師教學技能**，以及開發本專案所使用的**程式技能**。

---

## 🎓 學生學習技能

### 邏輯思維與問題解決
| 技能 | 說明 |
|------|------|
| 空間推理 | 在二維格狀空間中規劃路徑，辨識方向與距離 |
| 策略思考 | 預判死路，選擇最有可能通往終點的路線 |
| 試誤學習 | 走錯路後回溯，從失敗中調整策略 |
| 比較分析 | 比較 BFS、DFS、A★ 三種演算法的探索範圍與步數差異 |

### 資訊科學基礎概念
| 技能 | 說明 |
|------|------|
| 演算法概念 | 理解「有步驟的解題方法」就是演算法 |
| 圖形搜尋 | 理解迷宮可抽象為「節點＋邊」的圖形結構 |
| 最短路徑 | 理解 BFS「像水波擴散」必定找到最短路線的原理 |
| 座標系統 | 用（列, 行）表示位置，建立二維座標直觀概念 |
| 死路識別 | 理解「只有一個出口的路徑」即為死路（Dead End）|

### 數學基礎
| 技能 | 說明 |
|------|------|
| 格狀座標 | 在格狀平面中讀取與描述位置 |
| 計步概念 | 追蹤移動步數，比較實際步數與最佳步數 |
| 方向判斷 | 上下左右四個方向的空間方位感 |

## 🍎 教師教學技能

### 課堂應用方式

**1. 概念引入（5 分鐘）**
- 開啟老師模式，展示起始座標與最短路徑長度
- 提問：「老鼠在哪裡？用什麼方式描述它的位置？」

**2. 學生探索（15 分鐘）**
- 學生自行用鍵盤或觸控方向鍵操作
- 引導觀察：走了幾步？比最短路徑多了幾步？

**3. AI 演算法展示（10 分鐘）**
- 執行 BFS、DFS、A★，比較三者探索的格子數與路徑長度
- 學習角落：逐題展開說明，搭配課堂討論

**4. 進階挑戰（選用）**
- 切換至隨機迷宮生成器，選擇 16×16 困難難度
- 挑戰：AI 能否比學生自己走的路線更短？

### 對應課綱能力指標
- **計算思維**：分解問題、辨識模式、演算法設計
- **資訊科技應用**：操作互動程式，觀察程式行為與結果
- **數學領域**：空間感、座標概念、數量比較

## 💻 程式開發技能（開發者參考）

### 前端技術
| 技術 | 應用說明 |
|------|---------|
| React 18 | 函式元件、自訂 Hooks、Context-free 狀態管理 |
| TypeScript | 嚴格型別（`strict: true`）、泛型、聯合型別 |
| Vite 5 | 開發伺服器 HMR、生產打包、環境變數讀取 |
| CSS | Grid、Flexbox、CSS 自訂屬性、`@keyframes` 動畫 |

### 演算法實作
| 演算法 | 特性 |
|--------|------|
| BFS（廣度優先） | 保證最短路徑，`O(V+E)` 時間 / 空間複雜度 |
| DFS（深度優先） | 迭代堆疊實作，避免呼叫堆疊溢位 |
| A★ | Manhattan 距離啟發式，線性掃描開放集（適合小格） |
| 遞迴回溯法 | 迴圈式 DFS 生成迷宮，保證連通性 |

### React 設計模式
| 模式 | 說明 |
|------|------|
| Custom Hook | `useGameState`、`useAiSolver`、`useKeyboard` 分離邏輯 |
| Ref + State 搭配 | 用 `useRef` 同步追蹤位置，避免 StrictMode 雙重呼叫 |
| `useLayoutEffect` | 在瀏覽器繪製前同步更新，防止動畫閃爍 |
| `useCallback` 穩定化 | 防止鍵盤事件監聽器在每次渲染時重新註冊 |
| RAF 節流 | `requestAnimationFrame` 批次處理視窗縮放事件 |

### 部署
| 項目 | 說明 |
|------|------|
| GitHub Actions | push to `main` → 自動 build + deploy |
| Vite base path | `GITHUB_ACTIONS` 環境變數動態切換 `base` |
| Pages artifact | `actions/upload-pages-artifact@v3` 上傳 `dist/` |

